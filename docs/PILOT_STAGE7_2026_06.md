# Stage 7 학습자 파일럿 운영 가이드 — 2026-06-13 ~ 06-19

## 목적

7단계 검증 프로토콜의 마지막 단계. G3 (13개 단원 × 80+ 레슨)
검증 메타가 적용된 콘텐츠를 실제 학습자에게 노출해 다음을 확인:

1. **콘텐츠 정확성**: 정답/오답 분포가 의도된 난이도와 일치하는지
2. **misconception 탐지율**: `expected_errors`가 실제 학습자 오답과 매칭되는지
3. **PT(사전평가) 변별력**: 사전 vs 사후 점수 차이가 학습 효과를 반영하는지
4. **시간 분포**: 단계별 평균 소요 시간이 합리적인지
5. **버그/오타**: 학습자 피드백을 통한 콘텐츠 결함 발견

## 범위

- **학년**: G3 (3학년)
- **단원**: U1 ~ U13 (전체 13개)
- **레슨**: 모든 마이그레이션 완료 레슨 (80+)
- **단계**: PT(사전평가) → LEARN(개념) → TRY(시도) → R1/R2/R3(연습) → 사후평가
- **참여자**: 3-5명 학습자 (자발적 동의)
- **일정**: 2026-06-13(토) ~ 06-19(금), 7일간

## 측정 지표

### 1. 콘텐츠 정확성 (per problem)
- **이상 정답률**: PT(40-70%), TRY(60-85%), R1(75-95%), R3(60-90%)
- **이상치 탐지**: 정답률 <20% 또는 >95% → 콘텐츠 재검토 필요

### 2. Misconception 탐지율 (per misconception)
- 실제 오답이 `expected_errors`에 명시된 misconception과 매칭되는 비율
- 목표: 매칭률 ≥ 60%
- **v2.1 (2026-05-11 패치)**: 모든 CCSS-태그 항목(1,225개)에 `misconception_candidates: [3.NBT.A.2.M01, ...]` 부착. 진단 엔진은 오답 발생 시 항목의 candidate 리스트를 `backend/data/math/G3/misconceptions/{ccss}.json`과 조인해 후보 분류 가능. CCSS 표기 정규화 완료 (`3.NBT.1`→`3.NBT.A.1`, `3.NF.A.3.A`→`3.NF.A.3a` 등 7개 코드). 조인 검증: broken refs 0건.

### 3. 학습 효과 (per lesson)
- PT 점수 vs R3 점수 평균 차이
- 목표: R3 평균 ≥ PT 평균 + 20%pt

### 4. 시간 분포 (per stage)
- 단계별 평균 시간 (`time_spent_sec`)
- 이상치: 평균 ± 3σ 벗어나는 문항 → 재검토

### 5. 학습자 피드백 (정성)
- 매일 짧은 인터뷰 (5분): "오늘 어려웠던 부분?"
- 콘텐츠 결함 (오타, 모호한 표현, 잘못된 그림) 즉시 패치

## 합격 기준 (Stage 7 통과)

전체 G3 콘텐츠가 Stage 7 통과로 인정되려면:

- [ ] 80% 이상의 문항이 정답률 정상 범위 내
- [ ] 60% 이상의 misconception이 실제 매칭 발생
- [ ] 평균 학습 효과 ≥ +20%pt (PT → R3)
- [ ] 콘텐츠 결함 (오타·오류) 0건 (발견 시 즉시 패치)
- [ ] 학습자 만족도 ≥ 3.5/5 (5점 척도)

## 미달 시 대응

- **콘텐츠 정확성 미달**: 해당 문항 재제작 + Stage 1-6 재검증
- **misconception 매칭 미달**: `expected_errors` 보강 또는 misconception 파일 추가 항목 작성
- **학습 효과 미달**: LEARN 카드 보강 (개념 누락 가능성 점검)
- **만족도 미달**: UX/UI 개선 + 콘텐츠 재구성
